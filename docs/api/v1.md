# Soccer Tournament Dashboard · API Reference · v1

## Overview

This document defines the v1 REST API for the Soccer Tournament Dashboard.

All endpoints are prefixed with `/api/v1`. User-facing endpoints serve frontend page data. Admin endpoints are internal, protected by token auth, and used by manual refresh actions or scheduled background jobs.

Tournament-scoped resources are nested under `/tournaments/{tournament_id}`. Match details remain independent because a selected match has its own stable identity.

All error responses follow the standard format defined in [Error Format](#error-format).

---

## User-Facing Endpoints

### Health

| Method | Endpoint  | Description                                 |
| ------ | --------- | ------------------------------------------- |
| GET    | `/health` | Returns service, database, and cache health |

---

### Tournaments

| Method | Endpoint       | Description                            |
| ------ | -------------- | -------------------------------------- |
| GET    | `/tournaments` | Available tournaments for the selector |

---

### Tournament Standings

| Method | Endpoint                                 | Description                      |
| ------ | ---------------------------------------- | -------------------------------- |
| GET    | `/tournaments/{tournament_id}/standings` | Group standings for a tournament |

**Query parameters:**

| Param   | Type   | Required | Description                        |
| ------- | ------ | -------- | ---------------------------------- |
| `group` | string | No       | Filter to a single group, e.g. `A` |

---

### Tournament Matches

| Method | Endpoint                               | Description                  |
| ------ | -------------------------------------- | ---------------------------- |
| GET    | `/tournaments/{tournament_id}/matches` | All matches for a tournament |

---

### Match Details

| Method | Endpoint                     | Description                       |
| ------ | ---------------------------- | --------------------------------- |
| GET    | `/matches/{match_id}`        | Details for a single match        |
| GET    | `/matches/{match_id}/events` | Event timeline for a single match |

---

### Tournament Teams

| Method | Endpoint                             | Description                      |
| ------ | ------------------------------------ | -------------------------------- |
| GET    | `/tournaments/{tournament_id}/teams` | Teams registered in a tournament |

---

### Team Profile

| Method | Endpoint                                               | Description                            |
| ------ | ------------------------------------------------------ | -------------------------------------- |
| GET    | `/tournaments/{tournament_id}/teams/{team_id}/profile` | Team profile summary                   |
| GET    | `/tournaments/{tournament_id}/teams/{team_id}/matches` | Matches involving the selected team    |
| GET    | `/tournaments/{tournament_id}/teams/{team_id}/squad`   | Tournament squad for the selected team |

---

### Player Leaderboards

| Method | Endpoint                                           | Description                               |
| ------ | -------------------------------------------------- | ----------------------------------------- |
| GET    | `/tournaments/{tournament_id}/player-leaderboards` | Tournament player leaderboard by category |

**Query parameters:**

| Param      | Type   | Required | Description                           |
| ---------- | ------ | -------- | ------------------------------------- |
| `category` | string | Yes      | `goals`, `assists`, or `yellow_cards` |

---

### Knockout Bracket

| Method | Endpoint                               | Description                         |
| ------ | -------------------------------------- | ----------------------------------- |
| GET    | `/tournaments/{tournament_id}/bracket` | Knockout bracket rounds and matches |

---

## Admin Endpoints

All admin endpoints are under `/api/v1/admin` and require the `ADMIN_TOKEN`.

### Refresh Triggers

| Method | Endpoint                                         | Description                            |
| ------ | ------------------------------------------------ | -------------------------------------- |
| POST   | `/admin/tournaments/refresh-standings`           | Refresh standings from API-Football    |
| POST   | `/admin/tournaments/refresh-matches`             | Refresh matches from API-Football      |
| POST   | `/admin/tournaments/refresh-match-events`        | Refresh match events from API-Football |
| POST   | `/admin/tournaments/refresh-player-data`         | Refresh player and squad data          |
| POST   | `/admin/tournaments/refresh-player-leaderboards` | Refresh player leaderboard data        |

**Query parameters:**

| Endpoint              | Param         | Type    | Default | Description                             |
| --------------------- | ------------- | ------- | ------- | --------------------------------------- |
| `refresh-standings`   | `margin_days` | integer | `1`     | Days around tournament dates to include |
| `refresh-matches`     | `margin_days` | integer | `7`     | Days around tournament dates to include |
| `refresh-player-data` | `margin_days` | integer | `1`     | Days around tournament dates to include |

---

## Error Format

All errors return a consistent JSON shape:

```json
{
    "error": {
        "status": 404,
        "code": "NOT_FOUND",
        "message": "Tournament not found"
    }
}
```
